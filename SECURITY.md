# Security Policy

## Reporting

Do not publish secrets, credentials, private URLs, or exploit details in public issues.

If a security issue is found, contact the repository owner privately when possible or open a minimal public issue without sensitive details.

## Secret Handling

- Do not commit `.env` files, API keys, cookies, tokens, or private configuration.
- Rotate any secret that was accidentally committed or shared.
- Review diffs before opening pull requests.
